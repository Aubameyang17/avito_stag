Проект - создание автотестов для сервиса с данными по объявлениям

API-хост:https://qa-internship.avito.com
Тесты написаны на Python + pytest

Запуск тестов:
Выполнить из корня проекта:
pytest -q
pytest -vv (для подробного вывода)

Запуск одного конкретного теста:
pytest Test_api.py::test3_get_items_by_seller_positive -q

Что проверяют тесты:

Test1 — Создание объявления (ожидаем положительный результат)
POST /api/1/item
Обязательные поля: sellerID, name, price, а также statistics: {likes, viewCount, contacts}
Ожидается 200
В ответе ожидается поле status, из которого извлекается UUID созданного объявления

Test2 — Получение объявления по ID (ожидаем положительный результат)
GET /api/1/item/{id}
Ожидается 200
Проверяется, что объект с данным ID присутствует в ответе

Test3 — Получение всех объявлений продавца (ожидаем положительный результат)
GET /api/1/{sellerId}/item
Ожидается 200
Возвращается массив объявлений, все элементы должны содержать sellerId, совпадающий с отправленным

Test4 — Получение статистики объявления (ожидаем положительный результат)
GET /api/1/statistic/{id}
Ожидается 200
В ответе должны быть поля: likes, viewCount, contacts (числовые)

Test5 — Получение несуществующего объявления
GET /api/1/item/{id}
Ожидается 404

Test6 — Получение статистики несуществующего объявления
GET /api/1/statistic/{id}
Ожидается 404

Test7 — Получение объявлений по несуществующему sellerId
GET /api/1/{sellerId}/item
Ожидается 200 + пустой массив []
 
Test8 — Создание пустого объявления
POST /api/1/item с {}
Ожидается 400

Отладка и получение логов:
Если тест упал, в сообщении pytest выводятся:
фактический status code
тело ответа body: {...} — обычно содержит диагностическое сообщение
